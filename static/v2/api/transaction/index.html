Class Transaction

View code The transaction object is used to identify a running transaction. It is created by calling Sequelize.transaction().

To run a query under a transaction, you should pass the transaction in the options object.

ISOLATION_LEVELS

View code The possible isolations levels to use when starting a transaction

  READ_COMMITTED: "READ COMMITTED",

  SERIALIZABLE: "SERIALIZABLE"
}

LOCK

View code Possible options for row locking. Used in conjuction with find calls:

t1 // is a transaction
Model.findAll({
  where: ...
}, {
  transaction: t1,
  lock: t1.LOCK.UPDATE,
  lock: t1.LOCK.SHARE
})
